# Generate Plan

Действуй по методологии SDD. На основе моей активной спецификации составь детальный Implementation Plan.

Требования к плану:

- Декомпозиция: распиши конкретные шаги и их зависимости.
- Grounding: сверь план с текущим кодом. Укажи конкретные файлы, которые затронешь. Проверь, нет ли конфликтов с текущей архитектурой.
- Атомарность: сделай шаги такими, чтобы их можно было проверять по отдельности.

Спека:

Сохрани план в ту же директорию, где и спека, и дай имя `implementation-plan.md`.
