# Brief: Создание хедера/меню для сайта

**Related Issue:** [GitHub Issue #5](https://github.com/OlegPhenomenon/video-chat-and-translator/issues/5)
**Status:** draft
**Date:** 06.04.2026
**Owner:** Product/Tech Lead (для принятия решений по маршрутам и приоритету)

---

## Проблема

Сейчас в приложении нет единого хедера с навигацией, из-за чего пользовательский опыт «рвётся» между страницами:

- На разных экранах навигация реализована по-разному: на `Dashboard` меню находится внутри контента, на `Profile` — только ссылка «назад», на `Landing` — свой набор кнопок.
- Неавторизованный пользователь не видит постоянного списка CTA (Войти / Регистрация) в одном месте; кнопки разбросаны по экранам.
- Код дублируется: одна и та же ссылка/кнопка (например, выход) копируется на каждую страницу, что усложняет поддержку и делает изменения хрупкими.

**Измеримый результат:** 
- Навигация/кнопка выхода/CTA удалены из Dashboard.tsx, Landing.tsx, profile/Show.tsx (0 дублирований).
- Весь хедер и меню определяются в единственном месте (1 компонент).
- Хедер отображается на всех страницах приложения, меню переключается по статусу авторизации.

---

## Для кого / Стейкхолдеры

1. **Неавторизованные пользователи (гости)**
   - Приходят на публичную главную страницу, должны быстро найти ссылки на регистрацию и вход.
   - Находясь на страницах Devise, должны легко вернуться на главную.

2. **Авторизованные пользователи**
   - Перемещаются между дашбордом, профилем, главной.
   - Ожидают постоянную навигацию в хедере (как в типовом приложении), а не разрозненные кнопки на каждой странице.

3. **Команда разработки / Product Owner**
   - Нужна единая точка контроля для навигации, чтобы упростить добавление новых страниц и изменение меню.

---

## Контекст

Задача заведена как [GitHub Issue #5](https://github.com/OlegPhenomenon/video-chat-and-translator/issues/5).

Текущее состояние:
- Существуют роуты Devise (`/users/sign_in`, `/users/sign_up`) и профиль (`/users/profile`).
- Фронтенд на React + Inertia.js с CSS-фреймворком Tailwind.
- Навигация разбросана по разным компонентам страниц; отсутствует единый хедер.

**Маршруты:**
- `/` — публичная главная/лендинг (доступна всем: гостям и авторизованным).
- `/dashboard` — дашборд (доступен только авторизованным; для гостей редирект на `/`).
- `/users/profile` — профиль (доступен только авторизованным).
- `/users/sign_in`, `/users/sign_up` — страницы Devise (доступны гостям; если авторизованный откроет → редирект на `/dashboard`).

Зависимостей: нет; можно разработать независимо.

## Желаемый результат

**Поведение:**

Единый хедер отображается на всех страницах, меняя состав меню в зависимости от статуса авторизации.

**Меню для гостя (неавторизован):**
- `Зарегистрироваться` (ссылка на `/users/sign_up`)
- `Авторизоваться` (ссылка на `/users/sign_in`)
- `Главная` (ссылка на `/`) — **НЕ показывается на самой публичной главной `/`; показывается на страницах Devise.**

**Меню для авторизованного пользователя:**
- `Главная` (ссылка на публичную главную `/`)
- `Дашборд` (ссылка на `/dashboard`)
- `Профиль` (ссылка на `/users/profile`)
- `Выйти` (кнопка выхода из приложения)

**Критерии приёмки (поведение):**

- [ ] Хедер отображается на всех страницах приложения (публичных и защищённых).

- [ ] Меню содержит пункты:
  - Для гостя: "Зарегистрироваться", "Авторизоваться", "Главная" (но НЕ на самой публичной главной `/`).
  - Для авторизованного: "Главная", "Дашборд", "Профиль", "Выйти".

- [ ] Меню переключается в зависимости от статуса авторизации пользователя: если пользователь авторизован → меню авторизованного, иначе → меню гостя.

- [ ] Клики на пункты меню переводят на правильные страницы:
  - "Зарегистрироваться" → `/users/sign_up`
  - "Авторизоваться" → `/users/sign_in`
  - "Главная" → `/`
  - "Дашборд" → `/dashboard`
  - "Профиль" → `/users/profile`

- [ ] Клик на "Выйти" выполняет logout и переводит пользователя в состояние гостя (с отображением меню гостя).

- [ ] Переходы по меню происходят без полной перезагрузки страницы.

- [ ] Хедер согласован с системой дизайна приложения (Tailwind):
  - **Размеры:** высота хедера = `h-16` (64px на десктопе, `h-14` на мобильных < md);
  - **Отступы:** горизонтальные паддинги = `px-6` (десктоп), `px-4` (мобильный); вертикальные = `py-3`;
  - **Типография:** пункты меню = `text-sm`; активный/hover — более жирный шрифт или другой цвет;
  - **Цвета:** фон хедера = белый/светлый серый (согласно стилю Landing.tsx); текст = серый/тёмный; активная ссылка/кнопка — индиго (в стиле приложения);
  - **Брейкпоинты:** меню переходит в мобильное на `md` брейкпоинте (если нужно; детали в tech spec).
