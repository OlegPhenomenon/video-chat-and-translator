# Brief: AI-чат на странице видео

## Проблема

Пользователь, посмотрев загруженное видео и получив автоматический транскрипт (см. FT-017), не может задать AI-ассистенту вопрос по содержимому видео без выхода из приложения. Любой осмысленный анализ ролика — пересказ, поиск конкретного момента, проверка фактов, формулировка цитат, перевод фрагмента — приходится выполнять во внешнем чат-сервисе вручную: переключиться в браузерную вкладку, вставить транскрипт, сформулировать вопрос, скопировать ответ обратно. Это занимает минимум 4-5 ручных шагов на каждый вопрос и теряет привязку к видеоряду.

## Для кого

Авторизованный пользователь, который уже загрузил видео и (опционально) получил транскрипт через FT-017. Сценарий — анализ длинных лекций, интервью или учебных роликов: пользователь хочет получить смысловые ответы, не выходя из страницы видео.

## Контекст

После закрытия FT-017 (транскрипция) и FT-019 (панель субтитров) появилось текстовое представление содержимого видео, на котором стало возможно строить smart-функции. Issue #19 от 2026-04-14 — следующий шаг после транскрипта: использовать текст транскрипта как контекст для разговора с LLM. Без этого фичи транскрипции и субтитров остаются read-only, и пользователь не получает добавленной ценности от того, что транскрипт уже посчитан и хранится в браузере.

## Желаемый результат

На странице конкретного видео пользователь может задать произвольный вопрос про содержимое этого видео и получить связный ответ от выбранной им AI-модели. Минимально: один полный диалог из 3-5 реплик по содержимому ролика, без переключения вкладок и без копирования транскрипта руками. Выбор модели и собственного API-ключа остаётся за пользователем — приложение не несёт стоимости запросов.
