# FT-018: Показывать прогресс транскрибации

## What

### Problem

Сейчас при нажатии “Транскрибировать” пользователь не видит прогресса и не понимает, что происходит. Это приводит к путанице и ощущению, что приложение сломалось или «зависло».

Source: Issue [#22](https://github.com/OlegPhenomenon/video-chat-and-translator/issues/22).

### Outcome

| Metric ID | Metric | Baseline | Target | Measurement method |
| --- | --- | --- | --- | --- |
| `MET-01` | Доля попыток транскрибации без явного статуса/фидбека | Пользователь не видит прогресса | Во время транскрибации всегда виден статус/этап | Проверка по `SC-01` + `NEG-01` (UI evidence) |

### Scope

- `REQ-01` При запуске транскрибации UI сразу показывает явный статус “в процессе” (loading) и не оставляет кнопку/экран в неопределённом состоянии.
- `REQ-02` UI отображает прогресс/этапы настолько детально, насколько это возможно в текущей реализации (минимум: “подготовка” → “отправка/обработка” → “сохранение результата” → “готово”).
- `REQ-03` При ошибке транскрибации UI показывает понятное сообщение и возвращает пользователя в управляемое состояние (можно повторить попытку), без “вечного” лоадера.

### Non-Scope

- `NS-01` Не меняем провайдеров транскрибации, формат результата и контракт сохранения субтитров (это вне цели данной delivery-единицы).
- `NS-02` Не добавляем серверные endpoints/фоновые джобы для прогресса; прогресс отображается из client-side состояния.
- `NS-03` Не добавляем миграции БД и не меняем `schema.rb`.

### Constraints

- `CON-01` Реализация должна оставаться client-side и работать в devcontainer окружении без серверных операций.

## How

### Solution

Добавить на страницу видео понятный индикатор состояния транскрибации (loading + текст этапа + финальное состояние), который обновляется по мере выполнения шагов транскрибации и сохраняет/показывает ошибку при fail.

### Change Surface

| Surface | Type | Why it changes |
| --- | --- | --- |
| `video_chat_and_translator/app/frontend/pages/videos/Show.tsx` | code | Добавить отображение статуса/прогресса транскрибации рядом с контролами |
| `video_chat_and_translator/app/frontend/features/videos/transcription/*` | code | При необходимости — проброс/нормализация стадий выполнения, чтобы UI мог показывать этапы |

### Flow

1. Пользователь нажимает “Транскрибировать”.
2. UI сразу показывает “Транскрибация: в процессе” и текущий этап.
3. По завершении UI показывает “Готово” и возвращает контролы в нормальное состояние.
4. В случае ошибки UI показывает ошибку и позволяет повторить.

### Failure Modes

- `FM-01` Запрос к провайдеру падает (network/CORS/timeout) → UI показывает ошибку и предлагает повторить.
- `FM-02` Провайдер возвращает ошибку авторизации (401/403) → UI показывает ошибку и остаётся в управляемом состоянии.
- `FM-03` Транскрибация завершается, но сохранение/парсинг результата падает → UI показывает ошибку, не оставляя “вечный” loading.

## Verify

### Exit Criteria

- `EC-01` Во время транскрибации пользователь видит статус/прогресс и не остаётся без обратной связи.
- `EC-02` При ошибке отображается сообщение и UI не зависает в состоянии “в процессе”.

### Acceptance Scenarios

- `SC-01` Happy path: запустить транскрибацию → статус сразу становится “в процессе” → по завершении становится “готово”.
- `SC-02` Recovery happy path: после ошибки транскрибации пользователь нажимает повторить → транскрибация завершается успешно → статус становится “готово”.

### Negative / Edge Cases

- `NEG-01` Error path: симулировать ошибку транскрибации → статус показывает ошибку → можно повторить попытку.
