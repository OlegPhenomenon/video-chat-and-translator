# Brief: Создание страницы дашборда

**Related Issue:** [GitHub Issue #4](https://github.com/OlegPhenomenon/video-chat-and-translator/issues/4)
**Status:** draft
**Date:** 06.04.2026

### Проблема

После входа в систему авторизованный пользователь попадает на страницу Landing (`PagesController#index`), которая является маркетинговой заглушкой с описанием технологий стека. Это не подходящая точка входа для авторизованного пользователя — он видит рекламный контент вместо рабочего пространства. Отсутствие выделенного дашборда делает невозможным дальнейшее развитие продукта: некуда добавлять навигацию, статистику, историю сессий и прочие пользовательские функции.

### Для кого

Для зарегистрированных пользователей с подтверждённым email, которые только что вошли в систему или переходят на главную страницу приложения. Это основной экран, с которого пользователь будет начинать работу с сервисом.

### Контекст

Система авторизации реализована через Devise (фича [001-auth-email](../001-auth-email_COMPLETED/brief.md)), управление профилем доступно по маршруту `/users/profile` (фича [003-user-profile](../003-user-profile_COMPLETED/brief.md)). Аутентифицированный root уже определён в `routes.rb` и ведёт на `pages#index`. Фронтенд построен на React + Inertia.js (Vite + TypeScript + Tailwind CSS), компоненты страниц лежат в `app/frontend/pages/`. На данном этапе дашборд является заглушкой — полноценный функционал будет добавлен в последующих фичах.

### Желаемый результат

После авторизации пользователь попадает на аутентифицированный root (существующий маршрут `pages#index`) с моковым содержимым дашборда — заголовком «Dashboard» и базовой навигацией. Страница содержит ровно два элемента навигации: ссылку на `/users/profile` и кнопку выхода (Devise sign out). Страница является структурной основой для будущего функционала.
