## Контекст

В рамках Issue #6 IndexedDB-логика покрыта только ручной проверкой. Это осознанный риск. Для его закрытия необходимо:

1. Принять командное решение о добавлении frontend test harness (Vitest или Jest + fake-indexeddb).
2. Создать отдельный GitHub issue для Feature 5.3.

## Сценарии для покрытия (когда будет принято решение)

- Позитивный: `saveVideo` → `listVideos` → `findVideo` возвращает сохранённую запись
- `listVideos` после reload — записи персистируются
- `saveVideo` при заполненном хранилище → ошибка с кодом `quota_exceeded`
- `findVideo` несуществующего id → возвращает `null`/`undefined`
- Инициализация при отсутствии IndexedDB → ошибка с кодом `unsupported`

## Ручная проверка (обязательный минимум для Issue #6)

1. Загрузить видео → страница переходит на `/videos/:id`
2. Нажать reload → видео присутствует в списке на `/videos`
3. Открыть запись → видео воспроизводится без сетевого запроса к серверу
4. Проверить DevTools → ни один запрос на Rails-сервер с видеофайлом не уходит
5. Проверить DevTools → после ухода со страницы `/videos/:id` objectURL освобождён (отсутствует в памяти)
