# Product Requirements Document (PRD)

## Проблема / контекст
Языковой барьер затрудняет усвоение образовательного видеоконтента. Существующие решения либо дороги и требуют мощного бэкенда, либо не обеспечивают интерактивности. Необходима клиент-ориентированная платформа для видеоперевода, выполняющая ресурсоемкие ИИ-задачи (транскрибация, перевод, синтез речи) локально в браузере (ONNX WASM) или через API-ключи пользователя. Это кардинально снижает серверные расходы и повышает приватность.

## Цели
- **G01**: Реализовать локальный синхронный дубляж контента с устранением языкового барьера без тяжелого бэкенда.
- **G02**: Внедрить встроенного ИИ-тьютора с динамической песочницей (JS/React, Python) для генерации визуальных объяснений и анимаций «на лету».
- **G03**: Обеспечить бесшовный импорт видео (прямая загрузка файлов, браузерное расширение, "умная" работа с YouTube-iframe).
- **G04**: Интегрировать систему интервальных повторений (Anki) профильного материала через сопутствующего Telegram-бота.

## Целевая аудитория
- Обучающиеся, потребляющие англоязычный (или иной зарубежный) образовательный контент.
- Пользователи, нуждающиеся в глубоком интерактивном разборе сложных тем.
- Энтузиасты приватности, предпочитающие локальную обработку данных на своих устройствах.

## Ключевые сценарии
1. **Просмотр YouTube-видео**: Интеграция через стандартный iframe без скачивания видео (налету парсятся субтитры, поверх накладывается сгенерированная аудиодорожка).
2. **Взаимодействие с тьютором**: Запрос объяснения у ИИ-тьютора во время просмотра. Тьютор отвечает текстом/голосом и рендерит обучающую анимацию/график в встроенной песочнице.
3. **Импорт произвольного видео**: Перехват видеоролика на стороннем сайте с помощью браузерного расширения или ручная загрузка.
4. **Интервальное повторение**: Получение проверочных вопросов от Telegram-бота на основе сформированной сервером статистики просмотров.

## Что в MVP
- Локальный пайплайн обработки (транскрибация, перевод, TTS) через браузерные ONNX WASM-модели.
- Возможность подключения собственных API-ключей пользователей.
- Плеер-обертка для YouTube (наложение аудио поверх iframe).
- Базовый ИИ-тьютор с функциями конспектирования и ответов на вопросы.
- Песочница с поддержкой рендеринга (JS/React, базовый Python).
- Telegram-бот для интервальных повторений.
- Легковесный бэкенд для хранения прогресса, конспектов и аккаунтов.

## Что НЕ в MVP
- Серверное исполнение тяжелых ИИ-моделей (строго client-side или сторонние API ключи).
- Интеграция по Model Context Protocol для выгрузки баз знаний (Obsidian, Notion) — откладывается на следующие этапы, но потенциал заложен.
- Скачивание роликов YouTube на сервер.

## Ограничения и допущения
- **Производительность клиентского устройства**: Скорость локального инференса напрямую зависит от «железа» пользователя и поддержки WASM в браузере.
- **YouTube ToS**: Строгое соблюдение правил платформы — запрет на прямое скачивание потока (работа только через официальный iframe и парсинг субтитров).
- **Нагрузка на бэкенд**: Сервер обрабатывает только легковесные CRUD-операции (пользователи, статистика); исключается обработка медиафайлов.
