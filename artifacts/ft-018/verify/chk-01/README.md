## FT-018 — CHK-01 / EVID-01 (UI screenshots)

Цель: зафиксировать, что во время транскрибации пользователь **всегда** видит явный статус/этап, и при ошибке UI не “зависает” в вечном loading.

### Что нужно сохранить в эту папку

Положи сюда 3 скриншота (любые имена, но лучше по шаблону ниже):

- `01-in-progress.png` — после клика “Транскрибировать” виден stage `Подготовка` / `Обработка у провайдера` / `Сохранение результата`
- `02-success.png` — финальное состояние `Готово` (и “Готово: субтитры сохранены.”)
- `03-error.png` — состояние `Ошибка` или `Нужно действие` (validation_failed), при этом UI остаётся управляемым (кнопка доступна для повторной попытки/действия)

### Рекомендованный сценарий (ручной)

1. Открой “Видеотека”, зайди в любое видео.
2. В блоке “Транскрибация (через API)” введи API key (или намеренно неправильный — для error-state).
3. Нажми “Транскрибировать в .vtt”.
4. Сделай скриншот, пока виден in-progress stage.
5. Дождись завершения и сделай скриншот success **или** получи ошибку и сделай скриншот error.
6. Для `validation_failed`: убедись, что у видео уже есть субтитры и чекбокс “Перезаписать существующие субтитры” выключен, затем нажми “Транскрибировать…”.

### Почему не автоматизировали Playwright-мок (fallback `STOP-02`)

Авто-сценарий через Playwright требует детерминированно подготовить IndexedDB (видео должно существовать в библиотеке), а также поднять приложение в браузере. Это возможно, но требует отдельной инфраструктуры e2e-сетапа, которой сейчас нет “из коробки” в репозитории.
