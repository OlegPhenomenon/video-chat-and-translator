# Brief: Создание страницы дашборда

## Проблема

После входа в систему авторизованный пользователь попадает на маркетинговую страницу вместо рабочего пространства. Это не подходящая точка входа — он видит рекламный контент вместо инструментов для работы. Отсутствие выделенного дашборда делает невозможным дальнейшее развитие продукта: некуда добавлять навигацию, статистику, историю сессий и прочие пользовательские функции.

## Для кого

Для зарегистрированных пользователей с подтверждённым email, которые только что вошли в систему или переходят на главную страницу приложения. Это основной экран, с которого пользователь будет начинать работу с сервисом.

## Контекст

Базовая авторизация уже работает, профиль пользователя доступно. На данном этапе после входа пользователь попадает на маркетинговую заглушку — полноценный рабочий интерфейс отсутствует. Это важно сейчас, потому что без рабочего дашборда невозможно добавить ни одну пользовательскую функцию и начать основной сценарий использования сервиса.

## Желаемый результат

После авторизации пользователь попадает в рабочее пространство с базовой навигацией и возможностью перейти к управлению аккаунтом. Страница является структурной основой для будущего функционала.
