# Brief: Настройка CI/CD пайплайна

## Проблема

Сейчас нет автоматизированной проверки кода после изменений в репозитории. 100% проверок остаётся на локальных машинах разработчиков. Дефект может оказаться в основной ветке до того, как о нём узнает вся команда. Поставка в целевые окружения не зафиксирована как воспроизводимый процесс — нельзя заранее назвать, сколько ручных действий потребуется от мержа до выкладки. GitHub Actions workflow `.github/workflows/ci.yml` не существует, нет шагов `runs-on: ubuntu-latest` с `docker compose up` и `bundle exec rspec`.

## Для кого

Команда разработки и мейнтейнеры репозитория.

## Контекст

При росте числа изменений и участников ручная проверка масштабируются хуже, чем код. В проекте уже заложены контейнеризованная разработка и выкладка через Kamal — без оформленного контура проверки этот стек не даёт стабильной операционной дисциплины. Это блокер для привлечения новых контрибьюторов.

## Желаемый результат

При открытии и обновлении pull request, а также при слиянии в main автоматически выполняется набор проверок качества. Итог каждого прогона доступен всей команде. Путь от проверенного кода до развёртывания — один документированный процесс.
